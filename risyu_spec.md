# risyu 仕様書（リライト版）

カップル専用Webアプリ。より充実した毎日を送るためのサービスを提供する。

## 0. 今回のリライトの方針

- 現行実装（GitHub上のコード）から **スタンプカード機能** と **ビンゴ機能** を削除する。
- 画面構成を `risyu.txt` 記載の4画面に揃える：**ホーム / 支払い管理 / やりたいことリスト / マイページ**
- 「計画書機能」は `risyu.txt` には記載があるが未実装のため、新規に実装する。
- データベースは現行のまま（GAS スプレッドシート + localStorage キャッシュ、`auth.js` の仕組みを継続利用）。
- 不要になるファイル：`03_stamp.html`、`04_bingo.html`、およびこれらに紐づく `auth.js` 内の関数（`loadStampRemote`/`saveStampRemote`/`loadBingoRemote`/`saveBingoRemote` 等）、`navHTML` 内のスタンプ・ビンゴ導線。

## 2. ホーム画面

ダッシュボード形式。表示要素は以下3点（README.mdのスタンプ言及は削除し、risyu.txt準拠とする）。

1. **支払い差額サマリー**：両者の未払い金の差額を計算し、どちらが何円払うべきかを表示
2. **直近のやりたいことイベント**：やりたいことリストのうち、イベント日付が今日から2か月以内のものを期限が近い順に表示
3. **お知らせ**：各機能（支払い・やりたいこと）に新規追加や変更があった場合に通知表示

> スタンプ関連の通知・カードはホームから削除する。

## 3. 支払い管理機能

### 3.1 基本機能
- 両者の未払い金の履歴を閲覧・追加・削除できる
- データは**支払日（paidDate）から3か月経過すると自動削除**される
- タブで「自分」「相手」の支払い履歴を切り替え表示

### 3.2 データ項目
| 項目 | 説明 |
|---|---|
| 支払金額 | amount |
| ステータス | 未支払 / 一部支払 / 支払済 |
| 発生日時 | date |
| 支払日時 | paidDate |
| 支払経過履歴 | history（一部支払いの記録の配列） |
| メモ | memo |
| 支払者 | who（A/B） |

### 3.3 操作
- **新規作成（FAB）**：発生日時・金額・メモ・支払者・ステータスを入力して登録
- **差額支払い（FAB）**：相手と自分の未払い金の差額を自動計算し、どちらのユーザーか・支払った額・メモを入力して支払い処理。差額分だけ未払いデータを消化し、完済したものは「支払済」に、途中まで支払われたものは支払経過履歴に記録して残額を減らす
- **途中支払い記録**：個別の支払いデータに対し、一部支払いを記録（支払経過履歴に追加、残額を更新）

### 3.4 ガソリン代計算機能（画面右上から遷移/モーダル）
- 入力：走行距離(km)、今日のガソリン価格(円/L)
- 計算式：
  - 使用ガソリン量(L) = 走行距離 ÷ 15
  - 今回のガソリン代(円) = 使用ガソリン量 × 今日のガソリン価格
- 「請求」ボタンで支払い管理の未払いデータとして追加される

## 4. やりたいことリスト機能

### 4.1 基本機能
- 互いのやりたいことを追加・削除・閲覧できる
- データは**実行日(doneDate)から3年経過すると自動削除**される
- 検索・フィルター・並び替えが可能

### 4.2 データ項目
| 項目 | 説明 |
|---|---|
| やりたいこと（タイトル） | title |
| ステータス | 未実行 / 実行済み |
| 登録日時 | regDate（登録画面には表示せず、詳細画面でのみ確認可能） |
| 実行日付 | doneDate |
| イベント期間/日付 | period（デフォルトは単日。右端の＋ボタンで期間指定（開始日〜終了日）に切替可） |
| URL | url（右端の＋ボタンで複数追加可） |
| マップURL | map |
| 画像 | image（サイズ調整可） |
| 登録者 | registrar（A/B。詳細画面でのみ確認可能） |
| タグ | tags（ユーザー作成可、入力中に既存タグがドロップダウン候補表示） |
| やりたい度合いメーター | desire（自分用）/ desireB（相手用）。5段階。**自分のメーターのみ編集可能**。登録時必須ではなくデフォルト未選択 |
| 行動コスト | cost（⭐/⭐⭐/⭐⭐⭐＝簡単/普通/大変。二人で共有する1つの値。登録時必須ではなくデフォルト未選択） |
| メモ | memo |
| 計画書 | plan（4章参照。1項目につき0〜1件） |

### 4.3 表示・操作
- 円グラフ：①実行率（実行済み/全体）、②両者の総登録数の割合
- フィルター：ステータス・登録者・タグ（複数選択可）・行動コストで絞り込み、並び替え（登録日／実行日／やりたい度）も可能
- 一覧から項目をタップ→詳細モーダル表示→編集・削除・実行済み切替が可能

## 5. 計画書機能（新規実装）

### 5.1 位置づけ
- やりたいことリストの**1項目につき1計画書まで**作成可能（複数作成不可）
- 独立した画面ではなく、**やりたいこと詳細モーダルの中に計画書セクションとして埋め込む**
- 新規作成時はやりたいことリストの項目からタイトル・日付（イベント期間/日付）・URLを引用し、計画書側に複製する

### 5.2 データ項目
| 項目 | 入力形式 | 説明 |
|---|---|---|
| タイトル | 引用（編集可） | やりたいことのタイトルを引用 |
| 日付 | 引用（編集可） | イベント期間/日付を引用 |
| URL | 引用（編集可） | URLを引用 |
| スケジュール表 | 時刻＋内容のリスト | 例：「10:00 集合」「12:00 ランチ」のように行を複数追加 |
| もちもの | チェックリスト形式 | 項目を複数追加、チェックON/OFFで管理 |
| 出費 | 項目名＋金額のリスト | 複数項目を追加し合計金額を自動計算 |
| 買い出しリスト | チェックリスト形式 | 項目を複数追加、チェックON/OFFで管理 |
| メモ | テキストメモ | 自由記述 |
| 予約内容 | テキストメモ | ホテル・飲食店などの予約内容を自由記述 |
| やることリスト | チェックリスト形式 | 項目を複数追加、チェックON/OFFで管理 |

### 5.3 操作
- やりたいこと詳細モーダル内に「計画書」セクションを表示
  - 計画書が未作成の場合：「計画書を作成」ボタン→上記タイトル/日付/URLを引用した状態で作成フォームを開く
  - 計画書が作成済みの場合：計画書内容を表示し、編集・削除が可能
- 計画書の削除は、やりたいこと自体の削除とは独立して可能（やりたいことを削除した場合は計画書も連動して削除）

## 7. データ移行・互換性に関する注意

- 支払いデータ（`pay_v1` / GASシート `payments`）・やりたいことデータ（`wants_v1` / GASシート `wants`）・アカウントデータ（`accounts`）はそのまま継続利用する
- 新規追加：計画書データを保存するストア（例：`plans_v1` ローカル + GASシート `plans`、`wantsId` で `wants` と1:1で紐づける）
- 削除：スタンプデータ（`stamp_v1_*` / GASシート `stamp_*`）、ビンゴデータ（`bingo_v1` / GASシート `bingo`、`bingo_reward_by_count_v1`）は今後使用しない（必要であれば移行せず破棄）
- `auth.js` の `navHTML` からスタンプ・ビンゴの項目を削除し、4画面分のナビゲーションに変更する
